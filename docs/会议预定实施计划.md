# 会议室预定系统 — 两阶段实施计划

---

## 总体架构

```
┌─────────────────────────────────────────────────────────────┐
│                     Spring Boot 后端                         │
│  /api/meeting-room    会议室 CRUD                           │
│  /api/meeting-booking  预定 CRUD + 周期约 + 视图切换        │
│  /api/meeting-admin-lock  行政强征与释放                     │
│  Flyway 迁移: V14 + V15 建表                                │
└──────────────┬──────────────────────────────┬───────────────┘
               │                              │
    ┌──────────▼──────────┐      ┌───────────▼───────────┐
    │   Web 管理端 (Vue3)  │      │  小程序端 (原生 TS)    │
    │  上方: 会议室管理     │      │  底部Tab: 会议预定替换  │
    │  下方: 大看板使用页   │      │  登录统计              │
    └─────────────────────┘      └────────────────────────┘
```

## 阶段一：Web 端 + 后端接口（优先完成）

### 1.1 数据库设计（Flyway 迁移）

#### V14：会议室表 `meeting_room`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(32) PK | 雪花ID |
| name | VARCHAR(50) | 会议室名称（如"会议室 A (301)"） |
| location | VARCHAR(100) | 具体楼层位置 |
| capacity | INT | 座位数 |
| facilities | VARCHAR(500) | 配套硬件标签 JSON（如`["投影仪","白板","视频设备"]`） |
| photo_url | VARCHAR(500) | 实景照片 URL |
| is_active | TINYINT(1) | 1-上架 0-下架（下架后小程序不可见） |
| sort_order | INT | 排序权重 |
| create_time | DATETIME | 默认 CURRENT_TIMESTAMP |
| update_time | DATETIME | 默认 CURRENT_TIMESTAMP ON UPDATE |

#### V15：预定表 `meeting_booking`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(32) PK | 雪花ID |
| room_id | VARCHAR(32) FK | 关联会议室 |
| booking_date | DATE | 预定日期 |
| start_slot | INT | 起始时段编号（0=08:00, 1=08:30, ..., 19=17:30） |
| end_slot | INT | 结束时段编号（不包含，即[start_slot, end_slot)） |
| emp_no | VARCHAR(7) | 预定人工号 |
| emp_name | VARCHAR(50) | 预定人姓名（冗余，提高查询效率） |
| meeting_title | VARCHAR(200) | 会议名称（选填） |
| attendees | VARCHAR(500) | 参会人（选填） |
| is_weekly_fix | TINYINT(1) | 是否固定周期约 |
| group_id | VARCHAR(32) | 周期约组ID（同一批生成共享） |
| status | TINYINT | 1-正常 0-已取消 |
| create_time | DATETIME | 默认 CURRENT_TIMESTAMP |
| update_time | DATETIME | 默认 CURRENT_TIMESTAMP ON UPDATE |

> 索引：`idx_room_date (room_id, booking_date)`、`idx_group (group_id)`、`idx_emp_date (emp_no, booking_date)`

#### V16：行政征用表 `meeting_admin_lock`

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(32) PK | 雪花ID |
| room_id | VARCHAR(32) FK | 关联会议室 |
| lock_date | DATE | 征用日期 |
| start_slot | INT | 起始时段 |
| end_slot | INT | 结束时段 |
| reason | VARCHAR(500) | 征用原因 |
| dept_name | VARCHAR(100) | 发起部门 |
| operator_emp_no | VARCHAR(7) | 操作行政人员工号 |
| create_time | DATETIME | 默认 CURRENT_TIMESTAMP |

> 索引：`idx_lock_room_date (room_id, lock_date)`

### 1.2 后端 API 清单

#### 会议室管理 `/api/meeting-room`

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/` | 新增会议室 |
| `PUT` | `/{id}` | 编辑会议室 |
| `DELETE` | `/{id}` | 删除会议室 |
| `GET` | `/` | 查询所有会议室（含下架） |
| `GET` | `/active` | 查询上架会议室（小程序用） |
| `PUT` | `/{id}/toggle` | 上架/下架切换 |

#### 预定管理 `/api/meeting-booking`

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/` | 创建预定（含周期约拆分逻辑，单次/N条） |
| `PUT` | `/{id}` | 修改会议名称、参会人 |
| `DELETE` | `/{id}` | 取消单次预定 |
| `DELETE` | `/group/{groupId}` | 一键取消后续所有周期约 |
| `GET` | `/room/{roomId}` | 查询指定会议室某日所有预定（给前端看板用） |
| `GET` | `/room/{roomId}/week` | 查询指定会议室一周预定概览 |
| `GET` | `/my` | 查询当前用户所有有效预定 |

#### 行政征用 `/api/meeting-admin-lock`

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/` | 强制征用（覆盖会议室时段） |
| `DELETE` | `/{id}` | 释放征用（原预定恢复） |
| `GET` | `/room/{roomId}` | 查询指定会议室某日所有征用记录 |

#### 统一看板接口 `/api/meeting-booking/board`

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/board` | 查询某日所有会议室全局看板（合并预定+征用） |

参数：`?date=2026-07-09`

返回结构：
```json
{
  "rooms": [
    {
      "roomId": "...",
      "roomName": "会议室 A (301)",
      "capacity": 12,
      "facilities": ["投影仪", "白板"],
      "slots": [
        { "slot": 0, "type": "FREE" },
        { "slot": 1, "type": "BOOKED", "booking": { "empNo": "2036377", "empName": "张三", ... } },
        { "slot": 2, "type": "ADMIN_LOCK", "lock": { "reason": "集团高管会", ... } }
      ]
    }
  ]
}
```

> type 枚举：`FREE` / `BOOKED` / `ADMIN_LOCK`

#### 交互逻辑

1. **点击空白格子** → 弹出预定弹窗（会议名称、参会人、周期约开关）
2. **点击自己预约格子** → 弹出我的详情弹窗（可修改、可取消）
3. **点击他人预约格子** → 只读详情弹窗
4. **点击斜纹征用格子** → 只读征用详情
5. **管理员右键格子** → 强制征用弹窗

#### 状态区分

* `isAdmin`（工号判断，复用食堂模式）→ 展示管理区和右键菜单
* 表格中"自己的预定"：对比 `currentEmpNo` 与 `booking.empNo`

## 关键设计决策

1. **时间槽编号方案**：08:00=slot 0, 08:30=slot 1, ..., 17:30=slot 19（共20个半小时槽）
2. **周期约落库**：后端收到请求后直接拆分为 N 条独立记录，`group_id` 相同
3. **行政强征**：单独一张 `meeting_admin_lock` 表，覆盖层机制；释放=删除记录，原预定自动恢复
4. **管理员判断**：Web 端仅工号 `2036377` 可见管理区（与食堂菜单一致）
5. **过去时间判断**：以格子的结束时间对比当前系统时间
6. **小程序过去时间**：不做预约校验，前端直接比较时间判断是否可操作
