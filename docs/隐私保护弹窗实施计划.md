# 隐私保护弹窗实施计划

## 一、背景与目标

在小程序端实现隐私保护弹窗，满足微信小程序隐私合规要求，并在以下场景触发：

| 场景 | 入口 | 时机 |
|------|------|------|
| A | 首页 → 个人信息 tab → 点击用户卡片 | 进入个人信息页面之前 |
| B | 会议预约 → 信息不完善 → "去完善" | 跳转到个人信息页面之前 |
| C | 新用户首次登录 → profile-completion 弹窗 | 弹窗内嵌入隐私同意 |

## 二、设计决策

| # | 决策点 | 结论 |
|---|--------|------|
| 1 | 文案 | 简短提示语句 + 可点击跳转隐私政策全文页面 |
| 2 | 弹窗形式 | 自定义组件，使用 `<button open-type="agreePrivacyAuthorization">` |
| 3 | 持久化 | 依赖微信底层 `wx.getPrivacySetting`，无需本地 Storage |
| 4 | 新用户弹窗 | profile-completion 底部嵌入隐私同意按钮，一箭双雕 |
| 5 | 后端记录 | 同意后调 `POST /api/user/agree-privacy`，记录 `privacy_agreed_time` |

## 四、新增：`privacy-popup` 隐私授权弹窗组件

### 4.1 组件职责

可复用的隐私授权弹窗，在任意页面/组件中引入。

### 4.2 核心逻辑 (`privacy-popup.js`)

```
1. attached() 时调用 wx.getPrivacySetting
   - needAuthorization === false → 直接 triggerEvent('agree')，不弹窗
   - needAuthorization === true  → 展示弹窗
2. 用户点击 <button open-type="agreePrivacyAuthorization"> 同意
   → 触发 bindagreeprivacyauthorization 回调
   → 调用 POST /api/user/agree-privacy 记录同意时间
   → triggerEvent('agree')
   → 关闭弹窗
3. 用户点击"拒绝/暂不使用" → triggerEvent('refuse')
```

### 4.3 组件 properties

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `visible` | Boolean | false | 外部控制是否展示（用于手动控制展示时机的场景） |
| `autoCheck` | Boolean | true | 是否在 attached 时自动检查 wx.getPrivacySetting |

### 4.4 组件 events

| 事件 | 说明 |
|------|------|
| `agree` | 用户同意隐私协议（包括历史已同意无需弹窗的情况） |
| `refuse` | 用户拒绝 |

### 4.5 模板结构 (`privacy-popup.wxml`)

```html
<view class="privacy-popup" wx:if="{{ visible }}">
  <view class="mask"></view>
  <view class="dialog">
    <text class="title">隐私保护提示</text>
    <text class="desc">
      为了保障您的账户安全及提供会议室预定服务，
      我们需要收集您的头像、姓名及工号。
      在您使用前，请务必仔细阅读并同意
    </text>
    <!-- 可点击跳转全文 -->
    <text class="link" bindtap="goToPrivacy">
      《卡园健身打卡小程序用户隐私保护指引》
    </text>
    <text class="desc">
      如果您拒绝，将无法维护个人信息及预定会议室。
    </text>

    <view class="btns">
      <button class="btn-refuse" bindtap="handleRefuse">暂不使用</button>
      <button class="btn-agree" open-type="agreePrivacyAuthorization"
              bindagreeprivacyauthorization="handleAgree">
        同意并继续
      </button>
    </view>
  </view>
</view>
```

## 五、改造：`profile-completion` 新用户资料完善弹窗

### 5.1 当前状态

弹窗底部只有"开启体验"按钮，点击直接提交资料。

### 5.2 改造后

在"开启体验"按钮上方增加隐私协议勾选区域，将同意隐私与提交资料合二为一。

### 5.3 JS 改动 (`profile-completion.js`)

```javascript
data: {
  // ...原有 data
  privacyAgreed: false,   // 🆕 是否已同意隐私协议
},

// 🆕 隐私协议同意回调（由 open-type="agreePrivacyAuthorization" 触发）
handlePrivacyAgree: function (e) {
  // 同意后调后端记录
  var that = this
  request.post('/user/agree-privacy').then(function () {
    that.setData({ privacyAgreed: true })
  }).catch(function () {
    // 即使后端记录失败，微信侧已同意，仍放行
    that.setData({ privacyAgreed: true })
  })
},

// 🆕 跳转隐私全文
goToPrivacy: function () {
  wx.navigateTo({ url: '/pages/privacy/privacy' })
}
```

### 5.4 WXML 改动 (`profile-completion.wxml`)

在提交按钮上方插入：

```html
<!-- 隐私协议勾选区域 -->
<view class="privacy-check">
  <button class="privacy-agree-btn"
          open-type="agreePrivacyAuthorization"
          bindagreeprivacyauthorization="handlePrivacyAgree">
    <view class="checkbox {{ privacyAgreed ? 'checked' : '' }}">
      {{ privacyAgreed ? '✓' : '' }}
    </view>
    <text>我已阅读并同意</text>
  </button>
  <text class="privacy-link" bindtap="goToPrivacy">《隐私保护指引》</text>
</view>
```

### 5.5 提交按钮改动

`submitProfile` 方法开头增加判断：

```javascript
if (!this.data.privacyAgreed) {
  wx.showToast({ title: '请先阅读并同意隐私保护指引', icon: 'none' })
  return
}
```

## 六、改造：`pages/profile/profile` 个人信息页面

### 6.1 JSON 改动 (`profile.json`)

```json
{
  "navigationBarTitleText": "个人信息",
  "usingComponents": {
    "privacy-popup": "/components/privacy-popup/privacy-popup"
  }
}
```

### 6.2 JS 改动 (`profile.js`)

```javascript
data: {
  // 🆕 隐私弹窗相关
  privacyPassed: false,
  showPrivacyPopup: true,
  // ...原有 data
},

onLoad: function () {
  // 先不加载用户数据，等隐私弹窗结果
  // privacy-popup 组件会在 attached 时自动检查 wx.getPrivacySetting
  // 已同意过 → 直接触发 agree → onPrivacyAgreed → loadUser()
  // 未同意   → 展示弹窗 → 用户操作 → agree/refuse
},

onPrivacyAgreed: function () {
  this.setData({ privacyPassed: true, showPrivacyPopup: false })
  this.loadUser()
},

onPrivacyRefused: function () {
  this.setData({ privacyPassed: false, showPrivacyPopup: false })
  wx.showToast({ title: '需要同意隐私协议才能使用', icon: 'none' })
},

retryPrivacy: function () {
  this.setData({ showPrivacyPopup: true })
}
```

### 6.3 WXML 改动 (`profile.wxml`)

在页面顶部引入隐私弹窗组件，原有内容包裹在 `wx:if="{{ privacyPassed }}"` 中：

```html
<view class="profile-page">
  <!-- 🆕 隐私弹窗 -->
  <privacy-popup
    visible="{{ showPrivacyPopup }}"
    bind:agree="onPrivacyAgreed"
    bind:refuse="onPrivacyRefused"
  />

  <!-- 未通过隐私时展示提示 -->
  <view wx:if="{{ !privacyPassed && !showPrivacyPopup }}" class="privacy-blocked">
    <view class="blocked-icon">🔒</view>
    <text class="blocked-text">需要同意隐私保护协议后才能维护个人信息</text>
    <button class="retry-btn" bindtap="retryPrivacy">重新授权</button>
  </view>

  <!-- 原有内容 -->
  <view wx:if="{{ privacyPassed }}">
    <!-- 加载中 -->
    <view class="loading-area" wx:if="{{ loading }}">...</view>
    <!-- 主内容 -->
    <view class="profile-content" wx:if="{{ !loading }}">...原有表单...</view>
  </view>
</view>
```

## 七、后端改动

### 7.1 Flyway 迁移脚本 (`V30__add_privacy_agreed_time.sql`)

```sql
ALTER TABLE `user`
  ADD COLUMN `privacy_agreed_time` DATETIME NULL
  COMMENT '隐私协议同意时间' AFTER `update_time`;
```

### 7.2 新增接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/user/agree-privacy` | 记录当前用户的隐私同意时间 |

Controller 实现：

```java
@PostMapping("/agree-privacy")
public Result<Void> agreePrivacy() {
    String empNo = EmpContext.getEmpNo();
    if (empNo == null || "0000000".equals(empNo)) {
        return Result.fail("无法识别用户身份");
    }
    userService.lambdaUpdate()
        .set(User::getPrivacyAgreedTime, LocalDateTime.now())
        .eq(User::getEmpNo, empNo)
        .update();
    return Result.success();
}
```

## 十、注意事项

1. **微信隐私接口兼容性**：`wx.getPrivacySetting` 和 `open-type="agreePrivacyAuthorization"` 需要基础库 2.32.3+，当前小程序应满足此要求。
2. **微信后台配置**：需在微信公众平台「设置 → 服务内容声明 → 用户隐私保护指引」中配置与 `docs/隐私保护内容.md` 一致的隐私政策，否则 `agreePrivacyAuthorization` 按钮无效。
3. **后端接口容错**：`agree-privacy` 接口调用失败不应阻塞前端流程（见 `profile-completion.handlePrivacyAgree` 中的 `.catch` 处理）。
4. **`privacy-popup` 的 `autoCheck` 设计**：允许外部控制是否需要自动检查。在 profile 页面中使用默认值 true，未来如有特殊场景可通过 `autoCheck="{{ false }}"` 关闭自动检查并由外部手动控制。
