3.  **强化“后台接口前瞻性设计”：** 一期与二期的核心业务逻辑完全一致。后端接口将采用统一的 RESTful API 规范和 Token 鉴权机制，确保二期小程序端接入时，直接复用一期后台，实现低成本平滑升级。

# 📋 《Pocket Gym》（口袋健身）在线网页版 · 一期 PRD

## 一、 产品概述与定位

*   **产品名称：** Pocket Gym（口袋健身）
*   **产品形态：** 在线网页版（Web-PC 端体验优先）。
*   **核心定位：** 面向个人力量进阶、动态疲劳监控与在线社交 PK 的云同步健身管理系统。
*   **排期规划：**
    *   **一期（本期）：** 实现完整的 Web（网页版）功能，主打大屏高效排课、在线打卡结算、多维度复盘、全网实时榜单与在线 PK。
    *   **二期（规划）：** 新增微信小程序端。直接复用一期后端接口与数据库，仅做前端 UI 的移动端重构。

## 二、 系统架构与前瞻设计 (一期 & 二期共享)

由于一、二期共用一套后台，本期接口设计须满足以下统一规范：

```
                    ┌───────► 【一期前端：Web 网页端】(PC 布局)
                    │
【统一云端后台 API】─┼───────► 【二期前端：小程序端】(移动布局，直接复用 API)
                    │
                    └───────► 【统一云端数据库 (MySQL / PostgreSQL)】
```

*   **统一鉴权：** 采用 Token 鉴权机制，一期 Web 端支持账号/密码登录，并预留二期微信小程序 OpenID/UnionID 关联绑定的数据库字段。
*   **无状态接口：** 所有业务逻辑（如打卡结算、1RM计算、疲劳度消退）由后台 API 统一计算，保证两个前端表现一致。

## 三、 详细功能需求清单

### 1. 账户与云端档案模块 (User & Cloud Profile)

*   **1.1 用户登录与信息设置：**
    *   **登录方式：** 账号、密码在线注册/登录。
    *   **信息设置：** 支持设置昵称、上传头像。系统自动分配全网唯一用户 UID，用于后续的社交 PK 与榜单。
*   **1.2 身体指标记录看板（PC 大屏）：**
    *   **在线录入：** 录入每日体重（kg）、体脂率（%）。
    *   **双轴趋势图：** 在大屏右侧实时绘制并渲染长期体重与体脂波动的趋势折线图，支持自定义时间范围（如：近30天、近半年）。

### 2. 云端动作资产库 (Exercise Library)

*   **2.1 系统公共库：**
    *   系统预置标准力量训练动作目录。包含：动作名称、主攻肌群（如：胸大肌、三角肌）、参考预设组数/次数/重量。
*   **2.2 个人自定义动作（在线保存）：**
    *   用户可在线新建自定义动作，支持上传演示图片或视频的 URL 链接。动作保存后在个人资产库中可见。

### 3. 在线训练排课模块 (Routine & Planning)

在 PC 端大屏下，排课系统可以采用更加高效的**双栏式/分栏布局**：

*   **3.1 模板编排看板（分栏设计）：**
    *   **左栏：** 动作检索区。用户可按肌群、器械快速筛选动作。
    *   **右栏：** 模板编辑区。展示当前正在编辑的训练模板（如“推日计划”）。
    *   **挂载与微调：** 
        *   点击左栏动作即可添加至右栏模板中。
        *   可在右栏直接微调每一组的“目标重量”与“目标次数”。
    *   **排序微调：** 支持在右栏中通过上下箭头按钮直接调整动作的先后顺序。

### 4. 疲劳监控与智能选课 (Evaluate & Recommend)

*   **4.1 云端计算疲劳热力图：**
    *   **算法逻辑：** 系统调用当前用户在云端的历史打卡数据，计算各主要肌群距离上一次受到刺激的时间差（小时）：
        *   🔴 **极度疲劳（红色）：** 距上次训练 $< 24$ 小时。
        *   🟡 **恢复中（黄色）：** 距上次训练 $24 \sim 72$ 小时。
        *   🟢 **恢复充分（绿色）：** 距上次训练 $> 72$ 小时，或从未训练。
    *   **界面展示：** 在 PC 首页显要位置渲染人体肌肉热力图（SVG 交互图表），各肌群根据疲劳度渲染对应颜色。
*   **4.2 智能模板过滤：**
    *   点击热力图上的特定肌群（如：股四头肌），系统自动在下方检索并高亮推荐包含该肌群的训练模板。用户点击后可直接启动训练。

### 5. 现场带练与实时结算 (Execute & Submit)

*   **5.1 现场打卡工作台（PC 网页版）：**
    *   **历史对照：** 每个动作卡片内，同屏醒目展示该动作云端记录的“上次实际完成数据”，作为渐进超负荷的对照。
    *   **组数确认：** 采用列表勾选模式。每完成一组，点击“完成”复选框。
    *   **计时器：** 勾选后，页面顶部触发 60 秒组间休息倒计时，带有浏览器音频提示（倒计时结束时播放简易音效）。
    *   **现场微调：** 允许现场临时修改当前组的重量和次数，支持增加组、删除组。
*   **5.2 在线结算提交：**
    *   点击“完成训练”，数据实时同步至云端。
    *   **PR（历史纪录）提效：** 若检测到本组数据打破了该动作的历史记录，弹出 PR 庆祝弹窗，并询问：“是否将本次的实际重量同步至原模板？”。用户确认后，后台自动更新对应的模板参数。

### 6. 在线多人社交与 PK 模块 (Social & Gamification)

依托一期统一的云端后台，实现实时的全网/好友互动：

*   **6.1 在线双榜单：**
    *   **全网坚持榜（Consistency）：** 统计所有注册用户在“近 30 天内”在云端提交打卡的累计天数，由高到低排列。
    *   **力量进步榜（Progress）：** 
        *   计算用户指定三大项（深蹲、卧推、硬拉）的 **1RM（最大单次重量）近 30 天增长率**。
        *   榜单按增长率由高到低排列，鼓励进阶幅度最大的用户。
*   **6.2 实时力量 PK 墙：**
    *   **用户搜索：** 支持通过昵称或 UID 搜索全网其他在线用户。
    *   **同屏对比看板：** 选择目标用户后，在大屏上同屏对比双方的：
        1.  **双方历史总搬运吨位**；
        2.  **三大项最高重量直方图对比**；
        3.  **力量平衡五角星雷达图**（基于胸、背、腿、肩、核心五个部位的历史训练总容量比重）。

### 7. 课后复盘与分析预警 (Review & Analytics)

*   **7.1 多维度训练复盘：**
    *   **总容量趋势图：** 以时间为轴，绘制每周/每月训练总容量（搬运吨位）的柱状图或折线图。
    *   **频次贡献墙：** 类似 GitHub 贡献墙的绿色矩阵，直观反映全年的训练勤奋度。
*   **7.2 平台期/停滞预警：**
    *   若某动作数据在云端记录中连续 6 周的总容量或 1RM 无任何增长，在图表下方给予文字弱提示，建议更换动作模式。

## 四、 核心业务规则与算法定义

### 1. 1RM（最大单次重量）计算规则
用于评估力量进阶幅度，避免因组数次数变化无法直观对比：
$$\text{1RM} = w \times \left(1 + \frac{r}{30}\right)$$
*(其中 $w$ 为实际重量，$r$ 为实际次数)*

### 2. 单次训练总容量
$$\text{总容量（搬运吨位）} = \sum (\text{单组重量} \times \text{单组次数} \times \text{已完成组数})$$
