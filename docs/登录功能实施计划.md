## 一、需求分析

### 1.1 PRD 核心要点

| 维度 | Web 端 | 微信小程序端 |
|------|--------|-------------|
| 登录方式 | 用户名 + 密码 | 微信静默登录 (wx.login → code → openid) |
| 登录接口 | `POST /api/auth/web-login` | `POST /api/auth/wx-login` |
| 凭证 | `username` + `password` | 微信 `code` |
| 认证框架 | Sa-Token | Sa-Token |
| 新用户处理 | 管理员手动创建账号 | 静默注册 (status=0)，触发关键操作时弹出资料完善抽屉 |
| Token 传递 | `localStorage` → axios interceptor 自动注入 `satoken` 头 | `wx.setStorageSync` → request 封装注入 `satoken` 头 |

### 1.2 当前项目现状与差异分析

| 维度 | 当前状态 | 目标状态 | 差异 |
|------|---------|---------|------|
| 认证机制 | `X-Emp-No` 请求头 + `EmpContext` ThreadLocal | **Sa-Token** 统一鉴权 | 需要替换/融合 |
| 用户表 | **不存在**（仅有 `login_record` 日志表） | 新建 `user` 表 | 需新建 |
| 前端登录页面 | **无**，`App.vue` 直接渲染 Layout | 未登录时展示登录页 | 需新建 |
| 路由守卫 | **无** | 全局前置守卫 `beforeEach` | 需新建 |
| 小程序认证 | `empNo` 存本地，无实际登录 | 静默登录 + satoken | 需重写 |
| 小程序语言 | `.js` 文件（微信原生 JS） | **保持 `.js`**，不做 TS 迁移 | 无变更 |

### 1.3 关键架构决策（✅ 已确认）

**决策 1：Sa-Token 与 EmpContext 的融合方案**

Sa-Token 登录后通过 `StpUtil.getLoginIdAsLong()` 获取用户 ID（对应 `user.id`）。原有 `EmpContext.getEmpNo()` 改为从 Sa-Token 当前登录用户推导：

```
请求 → SaInterceptor 验证 satoken → 取出 userId → 查 user 表得 empNo → 注入 EmpContext
```

业务代码无需改动，`EmpContext.getEmpNo()` 调用继续有效。Web 端登录后自动使用该用户的 `emp_no` 字段（初始管理员：`2036377`），`UserSwitcher.vue` 组件保留。

**决策 2：统一 Token 模型**

Web 端和小程序端共用同一个 Sa-Token 体系，`StpUtil.login(userId)` 对两端完全一致。各端拿到 Token 后存入本地存储，后续请求统一携带 `satoken` 请求头。

**决策 3：微信 AppID / AppSecret（✅ 已提供真实值）**

| 配置项 | 值 |
|--------|-----|
| AppID（小程序ID） | `wx6bb890ad9c1e9c52` |
| AppSecret（小程序密钥） | `6e4c1ae7fcb99e203b13cd91c377a892` |

通过 `application.yml` 环境变量注入（不硬编码到仓库），本地开发可通过 `wx.mock` 开关控制 Mock/真实调用：

```yaml
wx:
  app-id: ${WX_APP_ID:wx6bb890ad9c1e9c52}
  app-secret: ${WX_APP_SECRET:6e4c1ae7fcb99e203b13cd91c377a892}
  mock: false   # false=真实微信调用；本地调试可临时设为 true
```

**决策 4：初始管理员账号（✅ 已确认）**

通过 Flyway 种子脚本 `V14__seed_admin_user.sql` 预创建：

| 字段 | 值 |
|------|-----|
| 用户名 | `admin` |
| 密码 | `admin123`（BCrypt 加密存储） |
| 工号 | `2036377` |
| 状态 | `1`（已完善） |

**决策 5：头像上传方案（✅ 已确认）**

上传到服务器本地磁盘，存储路径通过 `application.yml` 配置：

```yaml
app:
  upload:
    avatar-path: ${AVATAR_PATH:./uploads/avatar}
    avatar-url-prefix: /uploads/avatar
```

通过 `WebConfig` 注册静态资源映射使前端可直接访问。

**决策 6：小程序语言（✅ 已确认）**

微信小程序保持原生 JS 写法（`.js` 文件），不做 TypeScript 迁移。组件逻辑文件用 `.js` 而非 `.ts`。

#### 1.5 微信服务（支持 Mock / 真实双模式）

**文件：** [`java-fit-server/src/main/java/com/fit/service/WxService.java`](../java-fit-server/src/main/java/com/fit/service/WxService.java)

**文件：** [`java-fit-server/src/main/java/com/fit/service/impl/WxServiceImpl.java`](../java-fit-server/src/main/java/com/fit/service/impl/WxServiceImpl.java)

```java
@Slf4j
@Service
public class WxServiceImpl implements WxService {

    @Value("${wx.mock:false}")
    private boolean mock;

    @Value("${wx.app-id}")
    private String appId;

    @Value("${wx.app-secret}")
    private String appSecret;

    private final RestTemplate restTemplate = new RestTemplate();

    @Override
    public String getOpenIdByCode(String code) {
        if (mock) {
            log.warn("微信Mock模式：跳过真实code换取，返回占位openid");
            return "mock_openid_" + Math.abs(code.hashCode());
        }

        // 微信 code2Session 接口
        String url = String.format(
            "https://api.weixin.qq.com/sns/jscode2session?appid=%s&secret=%s&js_code=%s&grant_type=authorization_code",
            appId, appSecret, code
        );
        String resp = restTemplate.getForObject(url, String.class);
        // 解析 JSON 提取 openid（使用 Jackson ObjectMapper）
        // 检查 errcode，若不为 0 则抛异常
        log.info("微信 code2Session 成功, openid 已获取");
        return parseOpenId(resp);
    }

    private String parseOpenId(String resp) { /* Jackson 解析 */ }
}
```

> 微信 code2Session 返回格式：`{ "openid": "xxx", "session_key": "xxx", "errcode": 0 }`

#### 1.6 认证 Controller

**文件：** [`java-fit-server/src/main/java/com/fit/controller/AuthController.java`](../java-fit-server/src/main/java/com/fit/controller/AuthController.java)

| 接口 | 方法 | 入参 | 出参 |
|------|------|------|------|
| `/api/auth/web-login` | POST | `WebLoginDTO` | `Result<LoginResultDTO>`（含 token + userInfo[empNo]） |
| `/api/auth/wx-login` | POST | `WxLoginDTO` | `Result<LoginResultDTO>`（含 token + isNewUser + userInfo） |

**Web 登录核心逻辑**：
1. 根据 `username` 查 `user` 表
2. `BCrypt.checkpw(password, user.getPassword())` 校验
3. `StpUtil.login(user.getId())`
4. 返回 `{ token, userInfo: { empNo, nickname, ... } }`

**小程序登录核心逻辑**：
1. `wxService.getOpenIdByCode(code)` 获取 openid
2. 按 openid 查用户，不存在则新增（status=0, emp_no 自动生成）
3. `StpUtil.login(user.getId())`
4. 返回 `{ token, isNewUser, userInfo }`

### 阶段 3：微信小程序 —— 静默登录 & 资料完善

#### 3.1 静默登录流程

**文件：** [`mp-fit-ts/app.js`](../mp-fit-ts/app.js)（重写）

`onLaunch` 流程：
1. `wx.login({ success: (res) => { ... } })` 获取 `code`
2. `POST /api/auth/wx-login` 传入 `{ code }`
3. 成功后：
   - `wx.setStorageSync('satoken', res.data.token)`
   - `wx.setStorageSync('isNewUser', res.data.isNewUser)`
   - `wx.setStorageSync('userInfo', res.data.userInfo)`
4. 失败：静默降级，不阻塞小程序启动

#### 3.2 请求拦截器改造

**文件：** [`mp-fit-ts/utils/request.js`](../mp-fit-ts/utils/request.js)（修改）

- 401 响应处理：清除本地 token，触发重新静默登录
- 保留 `X-Emp-No` 注入

#### 3.4 首页拦截逻辑

**文件：** [`mp-fit-ts/pages/index/index.js`](../mp-fit-ts/pages/index/index.js)（修改）

`onLoad` / `onShow` 中：
1. 检查 `wx.getStorageSync('satoken')` 是否存在
2. 不存在 → 触发静默登录（调用 `app.js` 暴露的方法）
3. 存在 + `isNewUser === true` → 展示 `profile-completion` 组件
4. 首页内容正常渲染（新用户也能浏览）

#### 3.5 公共拦截方法

在 `app.js` 中暴露全局方法 `checkUserStatus(callback)`：
- 判断 `isNewUser`
- 新用户 → 弹出资料完善抽屉，完善后执行 `callback`
- 老用户 → 直接执行 `callback`

任何需要完整身份的操作（个人中心等）调用此方法进行前置拦截。

## 五、风险与注意事项

| # | 风险项 | 缓解措施 |
|---|--------|---------|
| 1 | Sa-Token 与 EmpContext 融合 | 在 SaInterceptor handle 中查 user 表推导 empNo 注入 EmpContext；`EmpInterceptor` 保留做降级 |
| 2 | 微信真实调用 | 已配置真实 AppID/AppSecret，`WxServiceImpl` 实现 `code2Session` 调用；本地调试通过 `wx.mock: true` 切回 Mock |
| 3 | 小程序保持 JS | 新建组件用 `.js` 文件；不引入 TS 构建链 |
| 4 | 头像上传路径 | `app.upload.avatar-path` 配置；`WebConfig` 注册 `file:` 协议静态资源映射；部署时确保目录可写 |
| 5 | Token 过期（30天） | 两端统一 401 → 清除 token → 重新登录（Web）/ 重新静默登录（小程序） |
| 6 | BCrypt 依赖冲突 | 仅引入 `spring-security-crypto`，不引入完整 Security Starter；必要时在 `@SpringBootApplication` 排除 `SecurityAutoConfiguration` |
| 7 | 管理员种子数据 | V14 脚本需预生成 BCrypt 密文；或用 `@PostConstruct` 在代码中检测并创建 |
