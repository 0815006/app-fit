# 健身打卡页面改造计划（小程序端）

> 目标：将小程序首页「健身动作」Tab 改造为「健身打卡」入口枢纽页，废弃原有动作浏览功能，通过 4 个入口卡片跳转到对应的功能页面。各功能页面独立置于 `components/workout/` 目录下，由轻量级 wrapper 页面承载。

---

## 一、改造总览

### 1.1 小程序 ↔ Web 端对应关系

### 3.2 设计原则

1. **Wrapper 页面极简**：每个 `pages/workout/*` 页面仅负责接收路由参数、引入核心组件、处理导航栏标题，**不包含业务逻辑**。
2. **核心逻辑在组件内**：所有数据加载、状态管理、用户交互均在 `components/workout/*` 中实现。
3. **组件通信**：Wrapper 页面通过 `properties` 向组件传递参数，组件通过 `triggerEvent` 向页面通知导航事件。
4. **复用性**：`action-picker` 组件被 `checkin-hub` 和 `makeup-form` 共用。

**组件**：[`components/workout/checkin-hub/`](mp-fit-ts/components/workout/checkin-hub/)

- **Props**：无
- **内部状态**：`dashboard`（看板数据）、`loading`、`selectedMuscleGroup`、`selectedMuscleCode`

`action-picker` 是一个通用的动作选择器组件，被 `checkin-hub`（今日打卡）和 `makeup-form`（补记录）共同使用。以底部弹出面板（Bottom Sheet）形式呈现。

**后端 API**：
| 接口 | 用途 |
|------|------|
| `GET /api/gym-action/all` | 获取全部动作 |

### 5.2 Props

| 属性 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `visible` | Boolean | `false` | 控制显示/隐藏 |
| `muscleGroup` | String | `''` | 限定大肌群（空=全部肌群） |
| `highlightMuscleCode` | String | `''` | 置顶高亮的二级肌肉编码 |

### 5.3 Events

| 事件 | 参数 | 说明 |
|------|------|------|
| `close` | — | 关闭弹窗 |
| `select` | `{ actionId, actionName }` | 用户选择了动作 |

## 九、关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 功能页位置 | `pages/workout/` + `components/workout/` 双层结构 | Wrapper 页面负责路由和导航栏，核心逻辑封装在组件中，便于复用和维护 |
| 后端接口 | **零新增**，复用全部已有 API | 后端接口已完整覆盖所有场景 |
| 动作选择器 | 提取为通用 `action-picker` 组件 | 今日打卡和补记录共用一个动作选择 UI |
| 计时训练 | 内嵌在 `checkin-hub` 组件中 | 微信小程序不支持 Web 端的独立弹窗模式，更适合内嵌子视图切换 |
| 首页健身 Tab 内容 | 完全替换为枢纽卡片（不保留原动作浏览） | 按需求明确"所有功能都抛弃" |
| `action-detail` 页面 | **保留不动** | 动作详情仍有参考价值，从 `action-picker` 中可跳转 |
| 样式方案 | 全自定义 WXSS，不引入 TDesign | 枢纽卡片+肌群卡片需要定制化布局，TDesign 组件在此场景不够灵活 |
